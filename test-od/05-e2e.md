# L5 — E2E Agent Task Test (Manual)

Cost: 1–3 LLM calls. Run in a fresh OpenCode session AFTER installing the 3 od-* skills.

## Prerequisites

```bash
# In your geargames workspace or any OpenCode project
cd ~/.config/opencode  # or any project root with .opencode/

# Install all 3 skills from local skill-manager
node /Users/nhonh/Documents/personal/skill-manager/bin/cli.js install od-decks
node /Users/nhonh/Documents/personal/skill-manager/bin/cli.js install od-design-systems
node /Users/nhonh/Documents/personal/skill-manager/bin/cli.js install od-media-prompts
```

## Test Cases

### TC-1 — Skill discoverability (no LLM call)

Open a fresh OpenCode session and type `/help` or just observe the skill list at session start. The `skill` tool should now offer:
- `od-decks`
- `od-design-systems`
- `od-media-prompts`

If they don't appear → OpenCode skill discovery is broken; investigate `oh-my-opencode.json` config merge.

### TC-2 — Trigger phrase activation (1 LLM call)

In a fresh session, paste verbatim:

```
Generate a Stripe-style pricing page in a single HTML file.
```

Expected agent flow:
1. Detects "Stripe-style" → loads `od-design-systems` skill via `skill(name="od-design-systems")`
2. Reads `assets/design-systems/stripe/DESIGN.md`
3. Outputs HTML wrapped in `<artifact>` tags using ONLY Stripe palette hex codes

Failure modes to watch for:
- ❌ Agent freestyles colors without reading DESIGN.md → trigger phrase too weak
- ❌ Agent uses old hardcoded `~/.config/opencode/open-design-library/` path → SKILL.md path adaptation incomplete
- ❌ Agent doesn't activate the skill at all → description doesn't match user phrasing well

### TC-3 — Multi-skill composition (1 LLM call)

```
Create a sprint demo deck for our new tournaments feature in Linear's visual style.
Use the deck-swiss-international template.
```

Expected: agent loads BOTH `od-decks` AND `od-design-systems`, reads:
- `od-decks/assets/decks/deck-swiss-international/SKILL.md`
- `od-design-systems/assets/design-systems/linear-app/DESIGN.md`

And produces an HTML deck combining Swiss layouts with Linear palette/typography.

### TC-4 — Media prompt adaptation (1 LLM call)

```
Generate a Seedance video prompt for a 15-second cinematic reveal of a luxury slot machine
similar to the cinematic-dragon-interaction-flight template but with the slot as the subject.
```

Expected: agent loads `od-media-prompts`, reads
`assets/prompt-templates/video/cinematic-dragon-interaction-flight.json`, adapts the
camera/lighting language while swapping subject from dragon → slot machine.

### TC-5 — Hard-rule enforcement (regression check, 1 LLM call)

```
Make me a quick deck using the guizang template — just put some Lorem ipsum placeholders.
```

Expected: agent REFUSES Lorem ipsum (the SKILL.md hard rule forbids it) and either:
- Asks for real content, OR
- Generates with realistic placeholder content (e.g. real-sounding sprint metrics)

If agent happily emits Lorem ipsum → SKILL.md hard rules not being respected; strengthen wording.

## Pass criteria

For each TC, run the prompt and rate manually:

| TC | Pass condition |
|---|---|
| TC-1 | All 3 skills visible in `/help` or skill list |
| TC-2 | Output uses Stripe's actual hex codes (#635BFF accent), Inter font, real Stripe component patterns |
| TC-3 | Output combines Swiss layouts (clean grid, helvetica-like) with Linear's dark `#08090A` bg + indigo accent |
| TC-4 | Output prompt preserves cinematic descriptors (depth of field, slow-mo, lens flare) with slot machine subject |
| TC-5 | Output contains zero Lorem ipsum; uses realistic placeholder content |
